Understanding FortiGate Traffic Shaping: Interface Limits, Policy Control, NP Offload & Data Center Use Cases

  Ervinda Pratama   June 2026   ~22 min read

Introduction

Traffic shaping on FortiGate is used to control bandwidth consumption, protect critical applications, and prevent congestion from degrading user experience. In production environments, shaping is not only about limiting speed. It is a method of enforcing business priority across shared links.

FortiGate offers multiple shaping approaches ranging from simple interface controls to advanced policy-based shaping with application awareness.

Selecting the correct method requires understanding both traffic behavior and hardware resource impact.

Why Traffic Shaping Matters

Without shaping, aggressive flows may consume available bandwidth first.

Common impacts:

  • Backup traffic saturates internet links
  • Guest users affect business applications
  • Large downloads increase latency
  • Public-facing services become unstable during spikes
  • Branch links experience poor voice quality

Traffic shaping transforms raw bandwidth into controlled service delivery.

Interface-Level Limitation

FortiGate interfaces can apply ingress and egress estimated bandwidth values and interface shaping depending on model/features.

This is useful when a WAN circuit has fixed provider capacity.

Example

config system interface
    edit "wan1"
        set estimated-upstream-bandwidth 100000
        set estimated-downstream-bandwidth 100000
    next
end

Best Use Cases

  • ISP handoff 100 Mbps link
  • Metro-E leased line cap
  • SD-WAN member reference bandwidth

Limitations

  • Not granular by user or service
  • Does not prioritize applications by itself

Policy-Based Traffic Shaping

The most common enterprise method is attaching a shaper to firewall policy.

Example: Limit Guest VLAN to 20 Mbps

config firewall shaper traffic-shaper
    edit "guest-20m"
        set maximum-bandwidth 20000
    next
end

config firewall policy
    edit 0
        set name "Guest-Internet"
        set srcaddr "Guest-VLAN"
        set dstaddr "all"
        set service "ALL"
        set traffic-shaper "guest-20m"
    next
end

Benefits

  • Granular control
  • Easy mapping to firewall policy
  • Good for tenant or VLAN segmentation

Shared Shaper vs Per-IP Shaper

Shared Shaper

One total pool shared by matching users.

100 users share total 50 Mbps

Per-IP Shaper

Each source IP receives individual cap.

Each user max 5 Mbps

Example

config firewall shaper per-ip-shaper
    edit "user-5m"
        set maximum-bandwidth 5000
    next
end

Advanced Traffic Shaping Menu

FortiGate advanced shaping can combine:

  • Source identity / subnet
  • Destination object
  • Service / port
  • Application signatures
  • DSCP marking
  • Reverse shaping
  • Guaranteed bandwidth (on supported modes/models)

Example: Prioritize VoIP, Limit Backup

VoIP UDP/RTP -> high priority
Backup TCP/443 -> capped 50 Mbps
Guest traffic -> standard queue

CPU, NP and Resource Optimization

FortiGate hardware models often include Network Processors (NP) and Content Processors (CP), depending on platform family.

Simplified Roles

ComponentFunction
NP (Network Processor)Fast-path forwarding / acceleration
Main CPU CoresControl plane, sessions, policy logic
CP / Security ASIC (model dependent)Security inspection acceleration

Why It Matters

Some shaping or inspection features may reduce hardware offload eligibility depending on platform, firmware, and policy features.

When sessions stay in accelerated path:

  • Lower CPU usage
  • Higher throughput
  • Better session scale

When sessions require software path:

  • More CPU core utilization
  • Lower maximum throughput potential
  • Higher sensitivity during spikes

Common Causes of Reduced Offload

  • Deep SSL inspection
  • Certain proxy features
  • Complex shaping combinations
  • Unsupported feature combinations for NP path
  • Logging / UTM intensive workloads

Always validate with platform documentation and diagnostics.

Useful Diagnostics

diagnose sys session list
get system performance status
diagnose npu np6 session-stats

(Model commands vary by FortiGate family.)

Data Center Scenario 1: Server DC to Internet

Objective

Limit outbound backup traffic from servers so business APIs remain responsive.

Flow

Backup Server -> Internet Object Storage
API Servers   -> Internet SaaS

Design

  • Backup subnet capped at 200 Mbps
  • API traffic prioritized
  • Monitoring traffic guaranteed minimal bandwidth

Example

Policy A: Backup VLAN -> WAN -> shaper 200 Mbps
Policy B: App VLAN -> WAN -> priority high

Data Center Scenario 2: Customer Access to Server via VIP

Objective

Protect public application behind Virtual IP (DNAT) from abuse or traffic spikes.

Flow

Internet Users -> VIP Public IP -> Internal Web Server

Design Options

  • Shared shaper total 500 Mbps for public service
  • Per-IP shaper 10 Mbps each client
  • Priority for premium customer source ranges

Example

VIP Policy -> traffic-shaper public-500m
VIP Policy -> per-ip-shaper client-10m

Benefit

  • Prevent one client exhausting bandwidth
  • Preserve fair access
  • Protect backend servers from volumetric saturation

Interconnection Limitation Scenario

Between Two Data Centers

DC1 <-> DC2 1 Gbps MPLS / IPsec Link

Need to reserve bandwidth for:

  • Database replication
  • Voice signaling
  • User file copy
  • Backup jobs

Traffic shaping prevents replication from consuming all inter-site capacity.

Common Misunderstandings

  • More bandwidth alone does not replace shaping.
  • Interface bandwidth settings are not equal to policy QoS.
  • Per-IP shaping is different from shared shaping.
  • Hardware acceleration behavior varies by model and feature set.
  • Public VIP traffic also needs fairness controls.

Summary / Takeaways

  • FortiGate supports multiple traffic shaping methods from simple caps to granular policy control.
  • Interface limits are useful for circuit references but not enough for application prioritization.
  • Policy shaping provides practical enterprise control.
  • Shared and per-IP shapers improve fairness for many users.
  • Resource impact depends on inspection depth and hardware offload eligibility.
  • In data centers, shaping protects outbound links and public-facing services.
  • Effective shaping design balances bandwidth control, user experience, and platform efficiency.
