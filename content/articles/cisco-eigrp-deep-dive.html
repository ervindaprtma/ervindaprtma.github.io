EIGRP Deep Dive: Metrics, Best Path Logic, K-Values & Multi-AS Design

  Ervinda Pratama   June 2026   ~18 min read

Introduction

Enhanced Interior Gateway Routing Protocol (EIGRP) is a high-performance dynamic routing protocol originally developed by Cisco. It combines characteristics of distance-vector and link-state approaches, which is why it is often described as an advanced distance-vector or hybrid routing protocol.

EIGRP became popular in enterprise networks because it offers fast convergence, efficient updates, unequal-cost load balancing, and simpler deployment compared to many traditional protocols.

Compared with earlier routing models, EIGRP gained strong adoption through fast convergence, operational simplicity, and efficient path calculation.

What is EIGRP?

EIGRP uses the DUAL (Diffusing Update Algorithm) to calculate loop-free paths and quickly react to topology changes.

How EIGRP Calculates Metric

By default, EIGRP uses:

  • Minimum bandwidth along path
  • Total cumulative delay along path

Classic metric formula (default K-values):

Metric = 256 * (Bandwidth + Delay)

Where:

Bandwidth = 10^7 / minimum bandwidth in Kbps
Delay = sum of delays in tens of microseconds

The scaling factor increases metric granularity and improves route comparison precision.

What Are K-Values?

K-values are weighting constants that determine which variables are included in metric calculation.

K ValueParameter
K1Bandwidth
K2Load
K3Delay
K4Reliability
K5Reliability modifier
K6Jitter / Extended metric (named mode modern enhancements)

Historically, classic EIGRP primarily uses K1 and K3 by default.

Default values:

K1=1 K2=0 K3=1 K4=0 K5=0

Meaning only bandwidth and delay affect route choice.

Why K-Values Exist

Different network environments may prioritize different path characteristics.

Examples:

  • WAN network may care about bandwidth.
  • Congested network may care about load.
  • Real-time network may care about delay/jitter.

K-values provide customizable metric weighting.

Important: All EIGRP neighbors in the same AS must use matching K-values or adjacency fails.

Full Composite Metric Formula

Classic formula:

Metric = 256 * [ (K1*Bandwidth) + (K2*Bandwidth)/(256-Load) + (K3*Delay) ] * [K5/(Reliability+K4)]

If K5 = 0, the reliability multiplier becomes 1.

Because default K2, K4, K5 are zero, the practical default formula simplifies greatly.

Feasible Distance vs Reported Distance

Feasible Distance (FD)

Lowest total metric from local router to destination.

Reported Distance (RD)

Metric advertised by neighbor to reach destination.

Feasibility Condition

A backup route becomes feasible successor if:

RD < current FD

This guarantees loop-free backup path.

Best Path Selection in Multi-Path Networks

When multiple routes exist, EIGRP evaluates:

  1. Lowest Feasible Distance
  2. Feasibility condition for backup paths
  3. Equal-cost path installation (if same metric)
  4. Unequal-cost load sharing with variance

Example

R1 reaches 10.10.10.0/24:

  • via R2 metric 1000
  • via R3 metric 1200

Best path = R2 (successor)
R3 may become feasible successor if condition is met.

Equal-Cost and Unequal-Cost Load Balancing

Equal Cost

Multiple same-metric routes can be installed.

Unequal Cost (Unique EIGRP Feature)

Use variance.

router eigrp 100
 variance 2

This may include routes up to 2x successor metric if eligible.

This is particularly effective for branch WAN environments with mixed circuit capacities.

EIGRP Autonomous System (AS)

The EIGRP AS number identifies routers participating in the same EIGRP domain.

Routers must match AS number to become neighbors.

Example:

  • R1 AS100 ↔ R2 AS100 = forms adjacency
  • R1 AS100 ↔ R3 AS200 = no adjacency directly

Multi-AS EIGRP Design

Organizations may use multiple EIGRP AS values for administrative separation.

Example Topology

Branch Sites (AS100) --- Core Router --- Data Center (AS200)

Core router runs both processes and redistributes routes.

Example Config

router eigrp 100
 network 10.1.0.0 0.0.255.255
 redistribute eigrp 200

router eigrp 200
 network 10.2.0.0 0.0.255.255
 redistribute eigrp 100

Route filtering and redistribution control are essential to prevent loops and route feedback.

Summary / Takeaways

  • EIGRP is a high-efficiency dynamic routing protocol using DUAL.
  • It runs over IP Protocol 88, not TCP/UDP ports.
  • Default metric uses bandwidth and delay.
  • K-values are weighting constants controlling metric inputs.
  • Successor is best path; feasible successor is instant backup path.
  • Variance enables unequal-cost load balancing.
  • Multi-AS deployments require redistribution and careful filtering.
  • EIGRP remains a capable routing option for Cisco-centric enterprise environments that value rapid convergence and flexible traffic engineering.
