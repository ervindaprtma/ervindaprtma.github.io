VLAN Deep Dive: Why VLAN Matters in Modern Switching

  Ervinda Pratama   June 2026   ~10 min read

Introduction

Early switching environments are often deployed as flat Layer 2 networks where every device shares the same broadcast domain. This model can function in small environments, but growth quickly introduces performance, security, and operational challenges.

VLAN (Virtual Local Area Network) addresses these limitations by dividing one physical switching infrastructure into multiple logical networks.

VLAN transforms basic connectivity into structured network architecture.

What is VLAN?

A VLAN is a logical segmentation method at Layer 2 that separates devices into different broadcast domains, even when they are connected to the same physical switch.

Devices in different VLANs cannot communicate directly unless routed through a Layer 3 device such as a router, firewall, or Layer 3 switch.

Concept Meaning
Physical Switch One hardware device
VLAN 10 Logical network for Finance
VLAN 20 Logical network for HR
VLAN 30 Logical network for Guest
Broadcast Domain Separate for each VLAN

Why VLAN is Needed

Without VLAN, every host is in the same Layer 2 segment.

Problems in flat networks:

  • Excessive broadcast traffic
  • Poor security separation
  • Difficult policy enforcement
  • Harder troubleshooting
  • No clean department/user segmentation

With VLAN, networks become structured.

Scenario Without VLAN With VLAN
Finance PC and Guest WiFi Same segment Isolated
Broadcast traffic Everywhere Limited per VLAN
Security policy Difficult Easy
Growth Messy Scalable

Basic VLAN Example

Consider a 24-port access switch:

  • Ports 1–8 → VLAN 10 (Finance)
  • Ports 9–16 → VLAN 20 (HR)
  • Ports 17–22 → VLAN 30 (Guest)
  • Port 24 → Trunk uplink to firewall/core switch

Although all endpoints connect to the same hardware platform, each VLAN operates as an independent Layer 2 segment.

Access Port vs Trunk Port

Port Type Function
Access Port Carries single VLAN for end devices
Trunk Port Carries multiple VLANs between switches / firewall / AP

Example:

# Cisco Style
interface g0/1
 switchport mode access
 switchport access vlan 10

interface g0/24
 switchport trunk encapsulation dot1q
 switchport mode trunk

VLAN Packet Flow (Frame Perspective)

When a PC in VLAN 10 sends traffic:

  1. PC creates Ethernet frame.
  2. Frame enters access port assigned to VLAN 10.
  3. Switch internally associates incoming traffic with VLAN 10.
  4. If exiting another access port in VLAN 10, frame is sent untagged.
  5. If exiting trunk port, switch inserts 802.1Q VLAN tag.
  6. Receiving switch reads tag and forwards inside same VLAN.

This mechanism allows the same VLAN to extend across multiple switches through trunk links.

Ethernet Frame with VLAN Tag

Standard Ethernet frame normally has:

Dst MAC | Src MAC | EtherType | Payload | FCS

With VLAN tagging (802.1Q):

Dst MAC | Src MAC | 802.1Q Tag | EtherType | Payload | FCS

The VLAN tag is inserted after Source MAC.

VLAN Tag Structure (4 Bytes Added)

802.1Q adds 4 bytes:

Field Size Purpose
TPID 2 Bytes Tag Protocol Identifier (0x8100)
TCI 2 Bytes Tag Control Information

TCI contains:

Subfield Bits Purpose
PCP 3 Priority (QoS)
DEI 1 Drop Eligible Indicator
VLAN ID 12 VLAN number (1–4094)

Packet Size Impact

Normal Ethernet frame MTU payload remains 1500 bytes, but tagged frame becomes larger on wire.

Frame Type Size
Standard Ethernet 1518 bytes
802.1Q Tagged 1522 bytes

Because VLAN adds 4 bytes. This overhead is operationally relevant for MTU-sensitive links, legacy devices, tunnels over Ethernet, or provider handoff designs.

VLAN Communication Example

PC-A in VLAN 10 wants to reach PC-B in VLAN 20. Direct Layer 2 switching cannot do this.

Traffic must go to:

  • Router-on-a-Stick
  • Layer 3 Switch SVI
  • Firewall Inter-VLAN Routing

Then security policy can be applied between VLANs.

Best Practice Design

Recommendation Why
Separate users by function Security + operations
Use trunk only where needed Reduce attack surface
Document VLAN IDs Easier operations
Reserve management VLAN Better control
Route through firewall for sensitive zones Visibility + policy

Summary / Takeaways

  • VLAN logically splits one switch into multiple Layer 2 networks.
  • Each VLAN is its own broadcast domain.
  • Access ports usually send untagged traffic; trunk ports carry tagged traffic.
  • 802.1Q inserts a 4-byte tag containing priority and VLAN ID.
  • Tagged Ethernet frames grow from 1518 to 1522 bytes.
  • VLAN is foundational for scalable campus, enterprise, and data center switching.

Understanding VLAN is a foundational step toward designing scalable, secure, and operationally efficient networks.
