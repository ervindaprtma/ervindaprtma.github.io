VXLAN Deep Dive: The Evolution Beyond Traditional VLAN

  Ervinda Pratama   June 2026   ~14 min read

Introduction

VLAN has been one of the most successful Layer 2 technologies in enterprise networking. It logically separates networks, reduces broadcast domains, and enables structured segmentation across switches.

However, modern infrastructure changed the game. Virtualization, cloud workloads, container platforms, and multi-site data centers introduced a scale problem that traditional VLAN design was never built to solve.

This is where VXLAN (Virtual Extensible LAN) emerged — a technology designed to extend Layer 2 networks across Layer 3 underlay networks while massively increasing segmentation scale.

VLAN solved campus segmentation. VXLAN solved data center mobility.

Why VLAN Became Limited

Traditional VLAN works well in campus and branch environments, but large-scale data centers exposed several limitations.

VLAN LimitationImpact
Maximum 4094 VLAN IDsLimited tenant/application segmentation
Layer 2 stretch complexityDifficult multi-rack / multi-site extension
Spanning Tree dependencyConvergence and blocked links
MAC table scaling pressureLarge broadcast domains
Poor workload mobilityVM migration challenges

As server virtualization increased, workloads needed to move between racks or clusters without changing IP addressing. VLAN-based architecture became operationally rigid.

What is VXLAN?

VXLAN stands for Virtual Extensible LAN.

It is an overlay technology that encapsulates Layer 2 Ethernet frames inside Layer 3 UDP packets, allowing Layer 2 segments to traverse routed IP networks.

Instead of relying on physical VLAN continuity, VXLAN creates logical segments across any IP fabric.

Why VXLAN Was Created

VXLAN was introduced to solve the rise of:

  • Large virtualized server farms
  • East-west traffic growth between workloads
  • Need for workload mobility between hosts/racks
  • Multi-tenant cloud environments
  • Massive segmentation requirements
  • Modern leaf-spine routed fabrics

Instead of stretching VLAN everywhere, networks moved to a routed underlay with overlay tunnels.

Basic VXLAN Traffic Flow

Example: VM-A wants to reach VM-B in same VXLAN segment.

  1. VM-A sends normal Ethernet frame.
  2. Leaf switch acting as VTEP receives frame.
  3. Switch maps VLAN to VNI.
  4. Original Ethernet frame is encapsulated into VXLAN/UDP/IP.
  5. Packet crosses routed underlay network.
  6. Remote VTEP decapsulates packet.
  7. Original Ethernet frame delivered to VM-B.

To end devices, it still feels like one Layer 2 network.

VXLAN Packet Structure

The original Ethernet frame remains intact as the inner payload while new transport headers are added for delivery across the routed underlay.

Original Ethernet frame is preserved and wrapped inside additional headers.

VXLAN Header Breakdown

VXLAN header is 8 bytes.

FieldSizePurpose
Flags8 bitsIndicates valid VNI
Reserved24 bitsReserved
VNI24 bitsVXLAN Network Identifier
Reserved8 bitsReserved

The VNI is the equivalent of VLAN ID, but far larger.

  • VLAN = 12-bit (4094 usable)
  • VXLAN = 24-bit (~16,777,216 segments)

Packet Size / MTU Impact

Encapsulation increases packet size, so MTU planning is a critical design requirement.

VXLAN adds overhead because of encapsulation.

Approximate added headers:

HeaderSize
Outer Ethernet14 bytes
Outer IP20 bytes (IPv4)
UDP8 bytes
VXLAN8 bytes
Total Additional Overhead~50 bytes

This is why data center fabrics commonly use jumbo MTU (e.g. 9000).

Common Misunderstandings

VXLAN does not eliminate the need for VLAN in most enterprise designs.

In many deployments, VLANs still exist at the access layer and are mapped to VNIs inside the fabric.

VXLAN is an overlay transport model, while VLAN often remains the local access segmentation method.

VXLAN also does not automatically solve poor underlay routing design. If the IP fabric is unstable, overlay performance will also suffer.

Think of VLAN as local edge segmentation, and VXLAN as scalable segmentation across routed fabrics.

Summary / Takeaways

  • VXLAN is the modern extension of VLAN concepts for scalable data center and cloud networking.
  • It encapsulates Layer 2 Ethernet frames inside UDP/IP packets, commonly using UDP port 4789.
  • VXLAN uses a 24-bit VNI, enabling approximately 16 million logical segments.
  • It solves many traditional VLAN scale and mobility limitations.
  • Best-fit environments include data centers, private cloud, multi-tenant hosting, and leaf-spine fabrics.
  • VLAN commonly remains at the access edge, while VXLAN extends segmentation across routed networks.
  • Successful VXLAN deployments depend on a stable underlay, proper MTU sizing, and a strong control plane such as EVPN.

Understanding VXLAN is the transition point from traditional switching to modern fabric engineering.
