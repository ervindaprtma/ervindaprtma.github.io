FortiGate SSL VPN Defense: Automation Stitch & Multi-Layer Security

  Ervinda Pratama   July 2026   ~18 min read

Overview

SSL VPN remains one of the most targeted enterprise services. Threat actors continuously scan for public-facing VPN portals to launch brute-force attacks, credential stuffing, and vulnerability exploitation. While standard password complexity and MFA are essential, they do not prevent the noise and resource consumption of persistent login attempts.

This article explores advanced defensive strategies using FortiGate Automation Stitches to automatically block brute-force attackers, combined with Geo-Location and ASN-based filtering to reduce the attack surface before authentication even begins.

Section 1 — Defensive Strategy: The "Block First" Mindset

Effective SSL VPN security should follow a layered approach:

  1. Surface Reduction: Block countries where you have no business presence (Geo-IP).
  2. Infrastructure Filtering: Block traffic from hosting providers, VPNs, and Tor exit nodes (ASN blocking).
  3. Rate Limiting: Limit how many login attempts can occur within a timeframe.
  4. Reactive Automation: Automatically ban IPs that trigger multiple failed logins (Automation Stitch).
  5. Authentication: Enforce strong MFA for all users.

Section 2 — Implementing Geo-IP and ASN Filtering

Most brute-force attacks originate from data centers (AWS, DigitalOcean, Linode) or specific geographic regions. We can use firewall address objects to block these proactively.

Geo-IP Blocking

config firewall address
    edit "COUNTRY_BLOCK_LIST"
        set type geography
        set country "CN" "RU" "IR" "KP"
    next
end

ASN Blocking (The Real Power Move)

Real users connect from residential ISPs. Attackers often connect from cloud infrastructure. By blocking ASNs of major cloud providers, you eliminate a significant portion of automated bot traffic.

config firewall address
    edit "ASN_BLOCK_LIST"
        set type asn
        set asn 16509 14061 15169  # Amazon, DigitalOcean, Google Cloud
    next
end

Section 3 — FortiGate Automation Stitch for Brute-Force Defense

Automation Stitches consist of a Trigger (an event in the system log) and an Action (a command or notification).

1. The Trigger: SSL VPN Login Failure

We look for the log event ID related to SSL VPN login failures.

config system automation-trigger
    edit "Trigger_SSLVPN_Fail"
        set event-type event-log
        set logid 39426  # SSL VPN login failed
    next
end

2. The Action: Ban IP

The action will execute a CLI script to add the offending IP to a quarantine list.

config system automation-action
    edit "Action_Ban_IP"
        set action-type cli-script
        set script "diagnose user quarantine add src4 %%log.remip%% 60 'Auto-ban by Automation Stitch'"
        set accprofile "super_admin"
    next
end

3. The Stitch: Combining Trigger and Action

config system automation-stitch
    edit "Stitch_SSLVPN_Defense"
        set trigger "Trigger_SSL_VPN_Login_Failed"
        set action "Action_Ban_IP"
    next
end

Section 4 — Monitoring and Verification

You can verify the active bans using the following CLI command:

diagnose user quarantine list

And check the automation execution history:

diagnose system automation history

Summary / Takeaways

  • Automation Stitches turn passive logs into active defense by reacting to threats in real-time.
  • ASN Blocking is often more effective than Geo-IP blocking because it targets the infrastructure used by botnets.
  • Layering Geo-IP, ASN, and Automation creates a "Hostile Environment" for attackers, forcing them to move to easier targets.
  • Always ensure your legitimate IP ranges are excluded from auto-ban scripts to prevent accidental lockout of valid employees.

Security is not a single feature; it is a continuous cycle of hardening, detecting, and responding.
