Detail: Outdoor Wireless Distribution (22 BTS Sites) with MikroTik & AirFiber

This project was implemented in phases, but can be summarized as follows:

Objective

Establishing a MAN-scale network as a high-speed distribution backbone for residential clients across villages in the Lamongan Regency area.

Implementation Workflow
  1. Defining the high-priority target coverage areas.
  2. Conducting comprehensive site surveys to analyze terrain elevation (contours), distances, and target bandwidth requirements.
  3. Surveying and selecting optimal geographical locations for Base Stations (Tower/BTS Antennas).
  4. Performing Link Planner analysis from the Central Tower to each site using GPS coordinates as the primary reference.
  5. Analyzing coverage area dynamics based on BTS coordinates and radio propagation patterns.
  6. Selecting specialized antenna hardware for Point-to-Point (PtP) and Point-to-Multipoint (PtMP) deployments.
  7. Physical hardware installation at the Main Tower and all 22 BTS branch sites.
  8. Configuring devices and establishing wireless links (antenna pointing, frequency scanning/selection, etc.).
  9. Fine-tuning every BTS backbone link to meet target bandwidth benchmarks and signal quality (CCQ/SNR).
  10. Conducting rigorous connectivity stress tests on all PtP links.
  11. Configuring Distribution Access Points (AP) at each BTS location.
  12. Performing on-site installations for residential clients using dedicated station radios.
  13. Individual tuning of client station links to ensure stable service delivery.
  14. Final end-to-end PtMP connectivity verification.
  15. Technical review and comprehensive project documentation.
  16. Implementing 24/7 monitoring and establishing troubleshooting protocols for backbone and user-level incidents.
Link Planning Tools
  • Conota Camera: For photo-documentation with integrated GPS coordinate tagging.
  • UISP Design Center: For detailed link planning and hardware specification modeling.
  • Google Earth Pro: For terrain contour analysis and line-of-sight (LoS) verification.
Detail: Local Network Monitoring System with MikroTik The Dude

This project was initiated to fulfill the requirement for a granular, locally-hosted network monitoring system.

Objective

Developing a centralized monitoring system for the entire network infrastructure using efficient protocols such as ICMP and SNMP.

Implementation Workflow
  1. Installation and hardening of the VMware Workstation host environment.
  2. Deployment and basic configuration of MikroTik Cloud Hosted Router (CHR).
  3. Activation of MikroTik P1 Level license for high-throughput management.
  4. Installation and initialization of The Dude server package.
  5. Design and creation of the logical Network Maps within the platform.
  6. Onboarding network devices for active health and service monitoring.
  7. Detailed device mapping based on the logical and physical topology.
  8. Granular management of monitored services (HTTP, DNS, PING, etc.).
  9. Integration of custom SNMP Walk and OID targeting for MikroTik-specific metrics.
  10. Organizing the monitoring dashboard using hierarchical Device Trees for better scalability.
Architecture & Stack
  • VMware Workstation 17: Virtualization platform for the MikroTik CHR instance.
  • MikroTik CHR File: Core operating system for the virtualized network controller.
  • The Dude Package: Primary service package for visual network management and alerting.
Detail: Dynamic Multi-Area Routing with OSPF Virtual-Links

Designing and implementing a scalable, high-convergence interior gateway protocol (IGP) for complex enterprise environments.

Objective

Establishing a robust OSPF architecture that supports multi-area scalability and redundant paths without routing loops.

Implementation Workflow
  1. Defining the OSPF Area hierarchy (Backbone Area 0 and Non-Backbone areas).
  2. Configuring Router IDs and Interface-specific OSPF parameters.
  3. Deploying OSPF Virtual-Links to bridge non-contiguous areas to the backbone instance.
  4. Implementing standard, stub, and NSSA (Not-So-Stubby Area) types for optimized LSA propagation.
  5. Configuring Inter-area route summarization (LSA Type 3) to reduce routing table size.
  6. Setting up MD5 authentication for secure routing adjacency between peers.
  7. Simulating link failures to test network convergence and DR/BDR election stability.
Core Technologies
  • MikroTik RouterOS v7: High-performance routing engine with updated OSPF logic.
  • GNS3 / EVE-NG: Network emulation for large-scale multi-area topology validation.

Key Implementation Steps
  1. Server Setup: Ubuntu-based LibreNMS installation with MariaDB and NGINX performance tuning.

FortiGate Next-Generation Firewall
Objective

Implementing a comprehensive security perimeter using FortiGate NGFW, covering SD-WAN, SSL-VPN, and server zone isolation.

Key Implementation Steps
  1. Zone Isolation: Creating dedicated security zones (DMZ, Internal, Servers) with strict firewall policies.
  2. SD-WAN: Configuring multi-WAN load balancing and SLA-based failover for critical business traffic.
  3. Remote Access: Setting up FortiClient SSL-VPN with multi-factor authentication (MFA) for secure management.
  4. IPS/IDS: Enabling Intrusion Prevention and Anti-Virus profiles to inspect encrypted traffic (Deep Inspection).
  5. Logging: Integrating FortiAnalyzer for centralized security reporting and threat hunting.
Core Features
  • Deep Packet Inspection (DPI): Inspecting SSL/TLS traffic for hidden threats.
  • Application Control: Granular control over web-based applications and services.
Advanced SLA Monitoring Stack
Objective

Developing a high-fidelity monitoring dashboard for SD-WAN performance metrics using the TIG (Telegraf, InfluxDB, Grafana) stack.

Key Implementation Steps
  1. Agent Setup: Configuring Telegraf SNMP inputs to poll FortiGate performance OIDs every 10 seconds.
  2. Data Storage: Indexing metrics into a time-series database (InfluxDB) for fast analytical queries.
  3. Visualization: Designing custom Grafana dashboards for latency, jitter, and packet loss visualization.
  4. Alerting: Setting up intelligent alert thresholds in Grafana to notify NOC teams via Telegram.
  5. Compliance: Generating weekly SLA reports for uptime and bandwidth utilization trends.
Architecture
  • Telegraf: Lightweight collection agent.
  • Grafana: Professional visualization and alerting platform.
  • InfluxDB: Performance-optimized time-series data storage.
