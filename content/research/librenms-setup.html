Setting Up LibreNMS from Scratch (Manual & Docker)

  Ervinda Pratama   March 2025   ~8 min read   Research Note

Overview

LibreNMS is a fully-featured open-source network monitoring system (NMS) built on top of SNMP, ping, and service checks. This research note documents the deployment of LibreNMS via Docker for monitoring an RT-RW Net ISP network serving 300+ customer endpoints using a mix of MikroTik (routers, managed switches, and wireless AP-ST units) and Ubiquiti (AirMax AP-CPE and EdgeRouter devices).

Environment

ComponentDetail

Deployment MethodDocker Compose (recommended for quick spin-up & portability)

Why Docker?

A manual LibreNMS installation requires configuring PHP, MySQL, Redis, RRDtool, nginx/apache, and multiple cron jobs across the OS. Docker Compose encapsulates all of these services into a single docker-compose.yml, making the setup reproducible and easier to update.

Docker deployment reduces setup time from ~2 hours (manual) to under 20 minutes, and makes upgrades a single docker compose pull command.

Step 1 — Install Docker on Ubuntu

# Update and install prerequisites
sudo apt update && sudo apt install -y ca-certificates curl gnupg

# Add Docker's official GPG key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Add Docker repository
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker Engine
sudo apt update && sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Add current user to docker group (avoids sudo for docker commands)
sudo usermod -aG docker $USER && newgrp docker

Step 2 — Clone LibreNMS Docker Repository

git clone https://github.com/librenms/docker.git librenms-docker
cd librenms-docker

Copy the example environment file and edit it:

cp .env.example .env
nano .env

Key variables to configure in .env:

# Database
MYSQL_DATABASE=librenms
MYSQL_USER=librenms
MYSQL_PASSWORD=StrongPasswordHere
MYSQL_ROOT_PASSWORD=EvenStrongerRootPassword

# LibreNMS app key — generate once
APP_KEY=base64:REPLACE_WITH_GENERATED_KEY

# Timezone
TZ=Asia/Jakarta

Step 3 — Start the Stack

# Start all services in the background
docker compose up -d

# Monitor startup logs
docker compose logs -f librenms

Wait for LibreNMS to fully initialize (typically 1–2 minutes). Access the web UI at http://<server-ip> and log in with the default admin credentials (set during first-run wizard).

Step 4 — Enable SNMP on MikroTik Devices

For LibreNMS to poll MikroTik devices, SNMP must be enabled and the community string configured. This can be done in bulk via a script pushed through MikroTik's API, or per-device via Winbox/CLI:

# On each MikroTik device
/snmp set enabled=yes contact="noc@yourisp.net" location="RT-XXX Node"
/snmp community add name=librenms-read addresses=192.168.100.0/24 security=none read-access=yes
✅ Tip: Use a dedicated SNMP read-only community string per location (e.g., rtrw-node-01) instead of a global public string. This makes it easier to rotate credentials without affecting all devices.

Step 5 — Enable SNMP on Ubiquiti Devices

# SSH into Ubiquiti device (EdgeOS / AirOS)
# For AirOS (UniFi/AirMax):
snmp.status=enabled
snmp.community=librenms-read
snmp.contact=noc@yourisp.net
snmp.location=Tower-A-Node
# Save and apply via /usr/etc/rc.d/rc.snmp restart

For EdgeRouter (EdgeOS), configure via: CLI → configure → set service snmp community librenms-read authorization ro

Step 6 — Add Devices to LibreNMS

Option A: Manual (Web UI)

Go to Devices → Add Device. Enter the device IP, SNMP version (v2c), and community string. LibreNMS will auto-discover the device type, model, and interfaces.

Option B: Bulk via CLI (for 300+ devices)

# Run inside the librenms container
docker exec -it librenms-docker-librenms-1 bash

# Add a single device
lnms device:add 192.168.1.1 -v v2c -c librenms-read

# Batch add from a list file
while IFS= read -r ip; do
    lnms device:add "$ip" -v v2c -c librenms-read
done < /tmp/device-list.txt

Step 7 — Configure Network Maps

LibreNMS has a built-in Network Map (using LLDP/CDP discovery) and a customizable Device Map. For the RT-RW Net topology:

  • Enable LLDP on MikroTik devices: /ip neighbor discovery-settings set discover-interface-list=all
  • Enable CDP on Ubiquiti devices where supported
  • Group devices by location (each RT node) using LibreNMS Device Groups
  • Use Custom Maps (Plugins → Maps) to lay out the star topology visually

Step 8 — Alerting

Configure alert rules for common ISP failure events:

Alert RuleTriggerSeverity
Device DownPing loss > 100% for 3 pollsCritical
Interface DownifOperStatus = downWarning
High Bandwidth UtilizationInterface in/out > 90% of speedWarning
SNMP No ResponseSNMP timeout after 3 retriesWarning

Alerts can be forwarded to Telegram (via webhook transport) for immediate NOC notification — ideal for a lean RT-RW Net operation.

⚠️ Known Issue: Ubiquiti AirMax devices often report incorrect interface speeds via SNMP (a known firmware bug). Manually set the interface speed in LibreNMS or use custom OIDs via the device override system.

Key Takeaways

  • Docker Compose is the recommended LibreNMS deployment method for ISP-scale environments — fast, reproducible, and upgrade-friendly.
  • MikroTik SNMP discovery works out of the box; Ubiquiti devices may need manual interface speed corrections.
  • Bulk device import via CLI script is essential when onboarding 300+ nodes — the web UI is too slow for this scale.
  • Use device groups by location/node to keep the dashboard manageable across a geographically distributed RT-RW network.
  • Telegram alerting provides near-real-time NOC notification without needing a dedicated paging system.
